I am a **Geospatial Algorithm Developer** and **Academic Researcher** bridging the gap between **forest biometrics** and **computer vision**. My work focuses on engineering automated pipelines for processing massive LiDAR point clouds (~10TB datasets) and multispectral imagery.

Currently, I am designing **deep learning architectures (U-Net variants)** for Land Use/Land Cover (LULC) classification and developing novel **segmentation algorithms** for individual tree detection in complex, multi-layered canopies.

* **Core Domain:** Computational Forestry, Remote Sensing, Forest Growth Modeling (FVS).
* **Infrastructure:** Cloud-native geospatial analysis (Azure/Docker), High-Performance Computing (HPC).

### üöÄ Featured Algorithms & Engineering

#### üå≤ [ISCL: Individual Tree Detection Algorithm](https://ergin-portfolio2.vercel.app/)
*Proprietary Algorithm (In Development, 2025)*
* **The Challenge:** Standard watershed and CHM-based algorithms fail to distinguish trees in overlapping, multi-layered canopy structures.
* **The Solution:** I engineered a novel approach utilizing **inter-slice clustering** and **geometric-graph fitting**. The algorithm analyzes vertical point density profiles to isolate individual trees without relying solely on surface models.
* **Stack:** Python (NumPy/SciPy), R, and C++ bindings for optimization.

#### üìä [National Forest Inventory Dashboard](https://tuoe.me/)
*Lead Architect & Full-Stack Developer*
* **System:** A decision-support system designed to visualize and analyze multi-year national inventory data for Turkey.
* **Engineering:** Built as a containerized **R Shiny** application deployed on **Microsoft Azure**.
* **Capabilities:**
    * Implemented distributed spatial querying for real-time analytics on national datasets.
    * Integrated interactive `Plotly` and `Leaflet` modules for dynamic data exploration.
    * Designed automated ETL pipelines to handle continuous data ingestion.
